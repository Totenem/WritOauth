# Chapter 5 – Artificial Intelligence Design

# 5.1 Artificial Intelligence Overview

WritOath employs a hybrid artificial intelligence architecture that combines deterministic stylometric analysis with Retrieval-Augmented Generation (RAG) to perform personalized authorship verification.

Unlike conventional AI-content detectors that attempt to classify whether a document was generated by artificial intelligence, WritOath evaluates whether a submitted paper is consistent with a student's established writing profile.

The system accomplishes this by retrieving verified historical writings, extracting measurable writing characteristics, and providing these as contextual knowledge to an open-source Large Language Model (LLM). This allows the model to reason using only trusted student-specific data rather than generalized knowledge learned during pre-training.

The architecture emphasizes explainability, reproducibility, and continuous profile evolution, ensuring that every analysis remains transparent and grounded in verifiable evidence.

# 5.3 Artificial Intelligence Workflow

The artificial intelligence workflow begins when a teacher uploads a student's written submission.

The AI Orchestrator coordinates the complete analysis process by invoking independent services responsible for feature extraction, semantic retrieval, language model inference, scoring, and explanation generation.

Rather than making an immediate judgment, the system first retrieves verified writing samples belonging to the same student. These documents become the reference context used by the language model during analysis.

The resulting evaluation is transformed into a structured report containing numerical consistency scores and natural-language explanations that assist teachers during decision-making.

---

# 5.4 Writing Fingerprint Service

The Writing Fingerprint Service is responsible for extracting measurable linguistic characteristics from every submitted document.

Unlike semantic embeddings, writing fingerprints represent deterministic features that can be directly compared across documents.

These features form the quantitative foundation of the authorship verification process.

The extracted features include:

| Category | Examples |
| --- | --- |
| Vocabulary Usage | Vocabulary richness, lexical diversity, uncommon word frequency |
| Sentence Structure | Average sentence length, sentence variation, complexity |
| Paragraph Structure | Average paragraph length, organization patterns |
| Grammar | Grammar consistency, passive voice frequency |
| Punctuation | Comma usage, semicolon frequency, quotation usage |
| Readability | Flesch Reading Ease, Grade Level |
| Formatting | Indentation, spacing, capitalization habits |
| Writing Habits | Transition phrases, discourse markers, repeated expressions |

These features remain consistent even when students write about different topics, making them valuable indicators of writing identity.

---

# 5.5 Embedding Generation

After feature extraction, the document is converted into a dense semantic embedding.

Embeddings encode the semantic meaning of a document into high-dimensional vectors that enable similarity comparison.

Unlike stylometric features, embeddings capture contextual relationships between ideas and writing content.

Each verified writing sample and submitted paper receives its own embedding before being stored or compared.

---

# 5.6 Retrieval-Augmented Generation (RAG)

Retrieval-Augmented Generation is the core intelligence mechanism of WritOath.

Instead of allowing the language model to rely solely on its pre-trained knowledge, the system retrieves verified writing samples belonging to the student and injects them into the prompt before inference.

This approach ensures that every comparison is grounded in authentic student data.

The retrieval process follows four stages:

1. Generate the embedding of the submitted paper.
2. Search ChromaDB for the most semantically similar verified writing samples.
3. Retrieve the Top-k writing samples.
4. Supply the retrieved samples to the language model as contextual knowledge.

By restricting the model's knowledge source to verified student writings, WritOath minimizes hallucinations and improves the reliability of authorship analysis.

---

# 5.7 Why Retrieval-Augmented Generation Instead of Fine-Tuning?

Fine-tuning an LLM would require a large collection of labeled authorship data and substantial computational resources.

In contrast, Retrieval-Augmented Generation allows WritOath to personalize analysis without retraining the model.

The advantages include:

- No custom model training required.
- Lower infrastructure cost.
- Faster deployment.
- Easy profile updates.
- Improved explainability.
- Student-specific reasoning.

These characteristics make RAG particularly suitable for educational environments where writing profiles continuously evolve.

# 5.8 LangChain Pipeline

LangChain serves as the orchestration framework responsible for implementing the Retrieval-Augmented Generation workflow.

Its responsibilities include:

- Managing document retrieval.
- Building structured prompts.
- Injecting retrieved writing samples.
- Calling the language model.
- Returning structured responses.
- Managing prompt templates.

LangChain simplifies the interaction between the retrieval layer and the language model while providing flexibility for future enhancements.

---

# 5.9 Prompt Engineering

Prompt engineering ensures that the language model produces structured, explainable, and reproducible outputs.

Instead of asking the model whether a paper is AI-generated, WritOath instructs the model to evaluate authorship consistency.

A typical prompt contains:

- Student writing profile summary.
- Retrieved verified writing samples.
- Extracted writing fingerprint.
- Submitted paper.
- Scoring instructions.
- Explanation format.

The model is instructed to produce structured JSON outputs, allowing downstream services to calculate scores and generate reports consistently.

---

# 5.10 Open-Source Language Model

The recommended language model for WritOath is **Qwen 2.5 Instruct**.

This model provides strong reasoning capabilities while remaining open-source and deployable on affordable infrastructure.

The language model does not determine whether a paper was generated by artificial intelligence.

Instead, it evaluates whether the writing style, linguistic characteristics, and overall expression remain consistent with the student's verified writing profile.

Because the model receives retrieved writing samples as context, its reasoning is grounded in evidence specific to the student rather than generalized language patterns.

---

# 5.11 Consistency Scoring Engine

The Consistency Scoring Engine transforms the language model's observations into quantitative metrics.

Instead of producing a binary decision, the engine generates a multidimensional assessment of writing consistency.

The evaluation may include:

| Metric | Description |
| --- | --- |
| Overall Consistency | Overall similarity to the student's writing profile |
| Vocabulary Consistency | Stability of vocabulary usage |
| Sentence Structure | Similarity in sentence construction |
| Grammar Consistency | Consistency in grammatical patterns |
| Readability | Comparison of reading complexity |
| Style Consistency | Similarity of writing habits and stylistic characteristics |
| Confidence Level | Reliability of the overall analysis |

These scores assist teachers without replacing human judgment.

---

# 5.12 Explainability Engine

Transparency is one of the primary design goals of WritOath.

The Explainability Engine converts technical AI analysis into teacher-friendly observations.

Instead of reporting only numerical scores, the system explains:

- Which writing characteristics remained consistent.
- Which characteristics deviated.
- Possible reasons for deviations.
- Supporting evidence from historical writing.

This allows teachers to understand the reasoning behind every recommendation.

---

# 5.13 Student Writing Profile Engine

The Student Writing Profile Engine maintains an evolving representation of each student's writing identity.

The profile is initially established using three verified writing samples written on different topics.

As teachers validate future submissions, additional verified writings may be incorporated through versioning.

This enables the system to recognize natural improvements in writing while preserving historical writing patterns for future comparison.

Rather than maintaining a static baseline, WritOath continuously refines each student's profile under teacher supervision.

---

# 5.14 Human-in-the-Loop Learning

Unlike fully autonomous AI systems, WritOath incorporates teacher validation into the learning process.

Teachers remain responsible for confirming or rejecting AI-generated recommendations.

Validated submissions may be integrated into the student's writing profile, allowing future analyses to become more representative of the student's evolving writing style.

This collaborative approach combines machine intelligence with expert human judgment.

---

# 5.15 Why WritOath Is Different

Most existing AI detection systems ask a single question:

> **"Does this document appear to be AI-generated?"**
> 

WritOath asks a fundamentally different question:

> **"Does this document match the student's established writing identity?"**
> 

This distinction changes the role of artificial intelligence from generalized AI detection to personalized authorship verification.

By combining stylometric analysis, Retrieval-Augmented Generation, explainable language model reasoning, and teacher-guided profile evolution, WritOath provides a more contextual and transparent approach to evaluating academic authenticity.
