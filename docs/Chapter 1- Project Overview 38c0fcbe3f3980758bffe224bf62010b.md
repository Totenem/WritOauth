# Chapter 1- Project Overview

# 1.1 Executive Summary

**WritOath** is an AI-powered authorship verification platform designed to assist educators in evaluating the authenticity of student-written works. Instead of relying on generalized AI-content detectors, WritOath builds a unique writing profile for each student using verified writing samples and continuously compares future submissions against that profile.

The system combines stylometric feature extraction, Retrieval-Augmented Generation (RAG), semantic retrieval, and an open-source Large Language Model (LLM) to produce explainable consistency assessments. By using only the student's verified writing as the primary knowledge source, WritOath minimizes false positives commonly associated with traditional AI detection tools while providing educators with transparent, evidence-based insights.

---

# 1.2 Project Background

The rapid advancement of generative artificial intelligence has significantly changed how students complete written assignments. Modern language models are capable of producing essays that closely resemble human writing, making it increasingly difficult for educators to determine whether a submission genuinely reflects a student's own work.

Existing AI detection tools primarily evaluate whether a document resembles AI-generated text based on generalized language patterns. While useful in certain situations, these approaches often produce inconsistent results because they evaluate the document in isolation rather than considering the individual student's established writing style.

WritOath approaches this problem differently. Instead of attempting to determine whether a document was generated by artificial intelligence, the system evaluates whether a newly submitted paper is consistent with the student's historical writing profile. This personalized approach provides educators with a more contextual and explainable assessment of authorship.

---

# 1.3 Problem Statement

Traditional AI detection platforms face several limitations:

- They rely on generalized language patterns instead of the student's own writing history.
- They frequently produce false positives for highly proficient writers or non-native English speakers.
- They provide limited explanations regarding how conclusions are reached.
- They do not continuously learn from verified student submissions.

As generative AI becomes increasingly sophisticated, detecting AI-generated text alone is becoming less reliable. Educational institutions require a system that evaluates authorship consistency rather than attempting to identify AI usage directly.

---

# 1.4 Proposed Solution

WritOath introduces a student-centric authorship verification framework that builds an evolving writing profile for every student.

Each student begins by submitting three verified writing samples covering different topics. These samples serve as the foundation of the student's writing profile.

Whenever a new paper is submitted, the system:

1. Extracts measurable writing characteristics.
2. Generates semantic embeddings of the document.
3. Retrieves the student's most relevant verified writing samples using Retrieval-Augmented Generation (RAG).
4. Uses an open-source Large Language Model to compare the new paper against the retrieved writing samples.
5. Produces an explainable consistency score and detailed analysis.
6. Allows the teacher to validate the result, enabling the student's writing profile to improve over time.

Rather than asking:

> "Was this written by AI?"
> 

WritOath answers:

> "Is this writing consistent with this student's established writing identity?"
>

# 1.5 Why the Name "WritOath"

The name **WritOath** is derived from the words **"Write"** and **"Oath."**

- **Write** represents the act of producing original written work.
- **Oath** symbolizes integrity, honesty, and academic responsibility.

Together, the name reflects the system's core objective: promoting authentic authorship by verifying that submitted work genuinely represents the student's own writing style.

Rather than acting as a punishment tool, WritOath serves as a digital affirmation of writing authenticity, encouraging academic integrity while providing educators with trustworthy evidence during evaluation.

---

# 1.6 Project Objectives

## General Objective

To develop an explainable AI-powered authorship verification platform that evaluates the consistency of student-written submissions by comparing them against the student's verified writing profile.

---

## Specific Objectives

The project aims to:

- Develop a centralized platform for managing teachers, students, subjects, and written submissions.
- Establish a verified writing profile for each student using baseline writing samples.
- Extract stylometric features from student documents for quantitative analysis.
- Implement Retrieval-Augmented Generation (RAG) to retrieve relevant historical writing samples.
- Utilize an open-source Large Language Model to compare new submissions against retrieved baseline documents.
- Generate consistency scores accompanied by detailed explanations.
- Allow teachers to review, validate, and provide feedback on every analysis.
- Continuously improve each student's writing profile using teacher-validated submissions.

---

# 1.7 Scope

The initial version of WritOath focuses on classroom-level authorship verification.

The system will support:

- Teacher authentication and profile management.
- Student management.
- Subject management.
- Uploading verified writing samples.
- Uploading new student papers.
- AI-powered writing comparison.
- Consistency scoring.
- Explainable analysis reports.
- Teacher validation.
- Continuous writing profile updates.

The system is designed to support approximately:

- 5–10 teachers
- 100–200 students

making it suitable for pilot deployments and educational research.

---

# 1.8 Limitations

The current version of WritOath will not:

- Determine with absolute certainty whether a document was generated by AI.
- Replace teacher judgment during academic evaluations.
- Train custom language models.
- Retrieve information from external internet sources.
- Support institution-wide learning management system integrations during the initial release.

Instead, all AI analysis is performed using only the student's verified writing samples and teacher-approved submissions.

---

# 1.9 Target Users

The primary users of WritOath include:

**Teachers**

Teachers upload verified writing samples, submit new student papers for analysis, review AI-generated reports, and validate system results.

**Students**

Students contribute verified writing samples that establish their writing profile and submit written assignments for evaluation.

**Educational Institutions**

Schools may utilize WritOath as a supplementary academic integrity tool that supports teachers in identifying inconsistencies in student writing while maintaining transparency and fairness.

---

# 1.10 Expected Impact

WritOath aims to redefine how academic integrity is evaluated by shifting the focus from generalized AI detection toward personalized authorship verification.

The system is expected to:

- Reduce reliance on unreliable AI detection tools.
- Provide teachers with explainable and evidence-based assessments.
- Encourage students to maintain authentic writing practices.
- Continuously improve writing profiles through validated submissions.
- Establish a scalable framework for future educational integrity systems.
